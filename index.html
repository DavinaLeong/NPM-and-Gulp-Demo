1.1. Intro and Trivia

NPM stands for Node Package Manager and it's used to maintain and update JavaScript packages. Bower is a common and popular alternative.

Gulp is a task runner which helps automate mundane and repetitve tasks like minifying CSS and JS scripts and compiling technologies like SASS and TypeScript. Grunt is another popular alternative to Gulp.

See Useful Links for more details if interested.

2.3. Installing NPM Packages

Make sure you are still in the static_repo folder in your terminal.

2.3.1. Installing a Package

Commands for installing a package:

Default npm install <package-name>
Specific version npm install <package-name>@<version.number>
Latest version npm install <package-name>@latest
Shorthand npm i <package-name>@latest

Let's try installing the lastest version of Bootstrap 3.

  Terminal
  • npm install --save bootstrap
By default, NPM installs the latest version of the package if it is not yet defined in package.json.
Otherwise, it will install the specified version; e.g. "bootstrap": "3.3.7". See Interpreting Version Numbers of package.json.

You will see a node_modules folder inside the static_repo folder. Inside the node_modules folder, you should see a boostrap folder.

WARNING: DO NOT commit the node_modules folder. Always remember to GIT-ignore this folder.

The node_modules folder contains alot of files and unnecessarily overloads your repo, increasing the time it takes to do GIT operations. Instead, ensure the package.json is updated with the necessary packages by using the --save and --save-dev flags and commit that. Then it's up to the individual developer to install the necessary NPM packages for their development. See --save and --save-dev Flags.

To find the NPM package's name of the resource, search for it on NPM's website.

2.4. Extra Reads

2.4.1. --save and --save-dev Flags

Quoted from NPM docs:

Dependencies are specified in a simple object that maps a package name to a version range. The version range is a string which has one or more space-separated descriptors. Dependencies can also be identified with a tarball or git URL.

Please do not put test harnesses or transpilers in your dependencies object. See devDependencies, below.

In other words, use the --save flag for packages necessary for the project run properly like jQuery and Bootstrap.

Use the --save-dev flag for packages necessary for the project's developement like task runners (Gulp/Grunt), test suites and precomilpers (SASS and TypeScript compilers).

2.4.2. Interpreting Version Numbers of package.json

No prefixes Install package at specified version of package; e.g. "bootstrap": "3.3.7".
Caret(^) Install major package version equal to or later than specified version; e.g. "bootstrap": "^3.3.7".
Tilde(~) Install minor package version equal to or later than specified version; e.g. "bootstrap": "~3.3.7"
Caret(^) vs Tilde(~)

Let's use an abitrary package Package XYZ as example and say it's at version 1.4.1. "package-xyz": "^1.4.1" will prompt NPM to install the latest release of Package XYZ. If version 2 comes out, NPM will install Package XYZ 2. "package-xyz": "~1.4.1" tells NPM to only install the lastest version of Package XYZ 1. It won't install Package XYZ 2.

2.4.3. Other Common NPM Commands

npm ls Lists all locally installed packages.
Add the -g flag to view globally installed packages.
Add the -l to list package details.
npm uninstall <package-name>
Shorthand: npm un <package-name>		 Uninstalls a package.
As usual, add the -g flag to uninstall a global package.
npm update Update local packages.

3.2.2. The copy-vendor Task


Bootstrap

Next, copy & paste these lines of code into the gulpfile.js and save.

3.2.3 The clear-vendor Task

Now we're going to make use of the Del package we imported earlier.

Copy & paste the following code just after the copy-vendor task.

gulp.task('clear-vendor', function() {
    del.sync([
        './vendor/bootstrap/**',
        '!./vendor/'
    ]);
});
PRACTICE: Figure out the command to execute this Gulp task and run it.

The order of the tasks does not matter to Gulp. It's more for readability than anything else.

What's the ! on line 4?

If you have some programming background, you'll know that ! means 'not'. Here, it means exclude a certain path. In other words, we're telling Del to exclude the vendors folder from deletion.

By default, Del will automatically remove folders if they are empty. This will cause Gulp to complain that the vendor folder can't be found when tasks are run.

Now run:

  Terminal
  • gulp clear-vendor

You should see the bootstrap folder gone from the static_repo folder. Since Bootstrap is gone, refreshing this page will return it to the original plain and ugly styles.

PRACTICE: Try to get Gulp to remove the font-awesome and jquery folders too.

Re-run gulp copy-vendor to restore the folders.

TIP: You can press the Up Arrow and Down Arrow keys to view previously run commands.

3.2.4. The update-vendor Task

Now let's create a 'super' task which will do the copying and deleting in one command.

Copy & paste this after the clear-vendor task:

gulp.task('update-vendor', ['clear-vendor', 'copy-vendor']);

Notice how there's no 'actual' code in this task? Also notice the array which contains the names of the 2 previously created tasks? Go ahead and execute the update-vendor task.

Did you see the clear-vendor and copy-vendor tasks being executed? The tasks in the array will be executed by Gulp before any tasks defined in update-vendor (if any).

Anyway, your gulpfile.js should now look like this:

var gulp = require('gulp');
var del = require('del');

gulp.task('copy-vendor', function() {
    // bootstrap
    gulp.src('./node_modules/bootstrap/dist/**')
        .pipe(gulp.dest('vendor/bootstrap'));
    console.log('~ copied bootstrap files');

    // jquery
    gulp.src('./node_modules/jquery/dist/**')
        .pipe(gulp.dest('vendor/jquery'));
    console.log('~ copied jquery files')

    // font awesome
    gulp.src([
        './node_modules/font-awesome/**/*.css',
        './node_modules/font-awesome/**/*-webfont.*'
    ])
        .pipe(gulp.dest('vendor/font-awesome'));
    console.log('~ copied font awesome files');
});

gulp.task('clear-vendor', function() {
    del.sync([
        './vendor/bootstrap/**',
        './vendor/font-awesome/**',
        './vendor/jquery/**',
        '!./vendor/'
    ]);
});

gulp.task('update-vendor', ['clear-vendor', 'copy-vendor']);

As usual, commit and push everything except the node_modules folder.

This section takes a deeper look into Gulp. It will run you through writing automated Gulp tasks using the watch function. Our goal is to merge all .css files into a single minified one.

4.1. Prepping

4.1.1. Clean Up Code

Before we start on the topic proper, let's clean up the code a bit.

Once again, open up your command line and navigate to your static_repo folder.

Let's create some constants to store the paths to our node_modules and vendor folders.

Open up your gulpfile.js and copy & paste this just under your require statememts:

const NODE_PATH = './node_modules/';

Now look through all your paths and modify them so they follow this pattern:

gulp.src(NODE_PATH + '/jquery/dist/**')

So that your gulpfile.js looks like this:

var gulp = require('gulp');
var del = require('del');

const NODE_PATH = './node_modules/';

gulp.task('copy-vendor', function() {
    // jquery
    gulp.src(NODE_PATH + 'jquery/dist/**')
        .pipe(gulp.dest('vendor/jquery'));
    console.log('~ copied jquery files')

    // bootstrap
    gulp.src(NODE_PATH + 'bootstrap/dist/**')
        .pipe(gulp.dest('vendor/bootstrap'));
    console.log('~ copied bootstrap files');

    // font awesome
    gulp.src([
        NODE_PATH + 'font-awesome/**/*.css',
        NODE_PATH + 'font-awesome/**/*-webfont.*'
    ])
        .pipe(gulp.dest('vendor/font-awesome'));
    console.log('~ copied font awesome files');
});

gulp.task('clear-vendor', function() {
    del.sync([
        './vendor/bootstrap/**',
        './vendor/font-awesome/**',
        './vendor/jquery/**',
        '!./vendor/'
    ]);
});

gulp.task('update-vendor', ['clear-vendor', 'copy-vendor']);

What's the point of this? We are simply extracting the path to the node_modules folder and storing it in a single variable so that if it ever changes ( though highly unlikely), we only have to change one location (the NODE_PATH constant) instead of multiple ones. Why const over var is out of the scope of this exercise.

Anyway, do the same for the vendor path; using VENDOR_PATH over NODE_PATH.

Your gulpfile.js should now look like this:

var gulp = require('gulp');
var del = require('del');

const NODE_PATH = './node_modules/';
const VENDOR_PATH = './vendor/';

gulp.task('copy-vendor', function() {
    // jquery
    gulp.src(NODE_PATH + 'jquery/dist/**')
        .pipe(gulp.dest(VENDOR_PATH + '/jquery'));
    console.log('~ copied jquery files')

    // bootstrap
    gulp.src(NODE_PATH + 'bootstrap/dist/**')
        .pipe(gulp.dest(VENDOR_PATH + '/bootstrap'));
    console.log('~ copied bootstrap files');

    // font awesome
    gulp.src([
        NODE_PATH + 'font-awesome/**/*.css',
        NODE_PATH + 'font-awesome/**/*-webfont.*'
    ])
        .pipe(gulp.dest(VENDOR_PATH + '/font-awesome'));
    console.log('~ copied font awesome files');
});

gulp.task('clear-vendor', function() {
    del.sync([
        VENDOR_PATH + 'bootstrap/**',
        VENDOR_PATH + 'font-awesome/**',
        VENDOR_PATH + 'jquery/**',
        '!' + VENDOR_PATH
    ]);
});

gulp.task('update-vendor', ['clear-vendor', 'copy-vendor']);

Run your Gulp tasks to make sure your code is working.

4.2.1 minify-css Task

Let's declare a const to store the path to the src folder. The compatibility: 'ie8' tells clean-css to generate CSS that's compatible with Internet Explorer 8 and earlier.

const SRC_PATH = './src/';

Now copy & paste the minify-css code above the copy-vendor task:

gulp.task('minify-css', function() {
    gulp.src([
        SRC_PATH + '**/*.css'
    ])
    .pipe(gulp_clean_css({compatibility: 'ie8'}))
    .pipe(gulp_concat('my-css.min.css'))
    .pipe(gulp.dest('dist'));
});

4.2. The watch Task

Now let's see how to automate the minify-css task using the Gulp watch function.

Copy & paste this code just above the minify-css task and execute it.

gulp.task('watch', function()
{
    gulp.watch(SRC_PATH + '**/*.css', ['minify-css']);
});

Technically, the task could be called any name–it's just a task name. But 'watch' is a standard convention name.

At this point, you might notice that you are unable run any commands on the command line as Gulp is holding it up. This is how you can use Gulp to help automate certain tasks like merging & compressing styles and scripts, run unit tests, etc.

Anyway, go ahead and change any of the .css files (except border-everything.css) in the src folder–e.g. change the background colour. Make sure to save your changes.

Did you notice the minify-css task being executed in the command line?

In the end, your gulpfile.js should look like this:

var gulp = require('gulp');
var del = require('del');
var gulp_clean_css = require('gulp-clean-css');
var gulp_concat = require('gulp-concat');

const NODE_PATH = './node_modules/';
const VENDOR_PATH = './vendor/';
const SRC_PATH = './src/';

gulp.task('watch', function()
{
    gulp.watch(SRC_PATH + '**/*.css', ['minify-css']);
});

gulp.task('minify-css', function() {
    gulp.src([
        SRC_PATH + '**/*.css',
        '!' + SRC_PATH + 'border-everything.css'
    ])
    .pipe(gulp_clean_css({compatibility: 'ie8'}))
    .pipe(gulp_concat('my-css.min.css'))
    .pipe(gulp.dest('dist'));
});

// --- manage vendor ---
gulp.task('copy-vendor', function() {
    // jquery
    gulp.src(NODE_PATH + 'jquery/dist/**')
        .pipe(gulp.dest(VENDOR_PATH + '/jquery'));
    console.log('~ copied jquery files')

    // bootstrap
    gulp.src(NODE_PATH + 'bootstrap/dist/**')
        .pipe(gulp.dest(VENDOR_PATH + '/bootstrap'));
    console.log('~ copied bootstrap files');

    // font awesome
    gulp.src([
        NODE_PATH + 'font-awesome/**/*.css',
        NODE_PATH + 'font-awesome/**/*-webfont.*'
    ])
        .pipe(gulp.dest(VENDOR_PATH + '/font-awesome'));
    console.log('~ copied font awesome files');
});

gulp.task('clear-vendor', function() {
    del.sync([
        VENDOR_PATH + 'bootstrap/**',
        VENDOR_PATH + 'font-awesome/**',
        VENDOR_PATH + 'jquery/**',
        '!' + VENDOR_PATH
    ]);
});
